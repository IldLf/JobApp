# Правила валидации полей в проекте

## Клиентская валидация

### 1. Регистрация и вход (`JobsLoginRegister.jsx`)

#### Форма регистрации (соискатель)

| Поле | Правило |
| ------ | --------- |
| `first_name` | Не пустое |
| `last_name` | Не пустое |
| `email` | Не пустой, содержит `@` и `.` |
| `password` | Минимум 8 символов |
| `confirmPassword` | Совпадает с `password` |

#### Дополнительная валидация для работодателя (модальное окно компании)

| Поле | Правило |
| ------ | --------- |
| `name` | Не пустое после trim |
| `city` | Выбран (не пусто) |
| `inn` | Обязателен, соответствует формату: 10 или 12 цифр (`/^\d{10}$|^\d{12}$/`) |

---

### 2. Форма создания/редактирования резюме (`ResumeForm.jsx`)

| Поле | Правило |
| ------ | --------- |
| `title` | Не пустое после trim |
| `profession_id` | Выбрано (не пусто) |
| `salary` | Если указано – должно быть числом ≥ 0 |

---

### 3. Форма создания/редактирования вакансии (`VacancyForm.jsx`)

| Поле | Правило |
| ------ | --------- |
| `title` | Не пустое |
| `description` | Не пустое |
| `profession_id` | Выбрано |
| `city` | Не пустое |
| `employment_type` | Выбрано |
| `experience_required` | Либо пусто, либо «Без опыта», либо соответствует формату `цифра-цифра года/лет/год` (например «1-3 года», «3-6 лет») |
| `salary_from` и `salary_to` | Если оба указаны, то `salary_from ≤ salary_to` |

---

### 4. Личный кабинет (`JobsAccount.jsx`)

#### Профиль соискателя / работодателя

| Поле | Правило |
| ------ | --------- |
| `phone` | Либо пусто, либо после удаления нецифровых символов длина 10 или 11 |
| `first_name` | Минимум 2 символа |
| `expected_salary` (соискатель) | Не больше 100 000 000 |
| `experience_years` (соискатель) | Не больше 70 |

#### Смена пароля

| Поле | Правило |
| ------ | --------- |
| `new_password` | Минимум 6 символов |
| `confirm_password` | Совпадает с `new_password` |

---

### 5. Отклик на вакансию / приглашение (`JobsCatalog.jsx`, `VacDetails.jsx`, `ResumeDetails.jsx`)

- Сопроводительное письмо (`cover_letter`) – не пустое.
- Для отклика на вакансию – обязательно выбран `resume_id` (одно из активных резюме пользователя).

---

## Серверная валидация (`server.js`)

### Регистрация (`POST /api/auth/register`)

- Email – проверка на уникальность (существующий пользователь).
- Если `user_type === 'employer'` и передан `company.inn` – проверка уникальности ИНН среди компаний.

### Создание резюме (`POST /api/resumes`)

- `title` – обязательное поле (если отсутствует → 400).

### Создание вакансии (`POST /api/vacancies`)

- `title` и `description` – обязательны (если отсутствуют → 400).

### Обновление статусов

- `PATCH /api/resume-responses/:id/status` – допустимые статусы: `pending`, `viewed`, `accepted`, `rejected`.
- `PATCH /api/vacancy-responses/:id/status` – допустимые статусы: `pending`, `viewed`, `accepted`, `rejected`.

### Смена пароля (`PUT /api/user/password/:userId`)

- Проверка, что переданный `current_password` совпадает с `password_hash` в БД (сравнение plain text, без хеширования).

### Отклики и приглашения

- Проверка на дублирование: нельзя повторно откликнуться на ту же вакансию; нельзя повторно отправить приглашение на то же резюме от одной компании.

### Остальные эндпоинты

- В основном проверяется существование запрашиваемых записей (пользователь, вакансия, резюме и т.д.). Явной валидации форматов полей (например, email, зарплата) на сервере нет – она полагается на клиент.
