# API Документация

Базовый URL: `http://localhost:5000`

Все ответы возвращаются в формате JSON.

---

## Общие эндпоинты

### `GET /api/health`

Проверка работоспособности сервера и подключения к БД.

**Ответ:**

```json
{
  "status": "OK"
}
```

---

### `POST /api/auth/login`

Вход пользователя.

**Тело запроса:**

```json
{
  "email": "user@example.com",
  "password": "plain_password"
}
```

**Успешный ответ (200):**

```json
{
  "success": true,
  "message": "Вход выполнен успешно",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "first_name": "Иван",
    "last_name": "Иванов",
    "user_type": "applicant",
    "is_active": 1,
    "full_name": "Иван Иванов"
  }
}
```

**Ошибки:**

- `401` – Неверный email или пароль.
- `500` – Ошибка сервера.

---

### `POST /api/auth/register`

Регистрация нового пользователя.

**Тело запроса:**

```json
{
  "email": "user@example.com",
  "password": "plain_password",
  "first_name": "Иван",
  "last_name": "Иванов",
  "user_type": "applicant|employer",
  "company": { // только для user_type = employer
    "name": "ООО Ромашка",
    "description": "...",
    "city": "Москва",
    "inn": "1234567890"
  }
}
```

**Успешный ответ (201):**

```json
{
  "success": true,
  "message": "Регистрация прошла успешно" | "Регистрация прошла успешно. Ожидайте подтверждения администратора.",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "first_name": "Иван",
    "last_name": "Иванов",
    "user_type": "applicant",
    "is_active": 1 | 2,
    "full_name": "Иван Иванов",
    "profile": { // для соискателя
      "id": 10,
      "type": "applicant"
    } // или для работодателя: { id, name, type: "employer" }
  }
}
```

**Ошибки:**

- `409` – Пользователь с таким email уже существует.
- `400` – Компания с таким ИНН уже зарегистрирована.
- `500` – Ошибка сервера.

## Резюме

### `GET /api/resumes`

Поиск и фильтрация резюме (активные, `is_active = 1`). Поддерживается пагинация.

**Query параметры:**

| Параметр | Тип | Описание |
| ---------- | ----- | ----------- |
| profession_id | int | ID профессии |
| experience | string | Фильтр по опыту (LIKE) |
| salary_from | int | Нижняя граница зарплаты |
| salary_to | int | Верхняя граница зарплаты |
| search | string | Поиск по заголовку, описанию, опыту, имени/фамилии, профессии |
| page | int | Номер страницы (default 1) |
| limit | int | Записей на странице (default 5) |

**Успешный ответ (200):**

```json
{
  "success": true,
  "resumes": [
    {
      "id": 10,
      "title": "Frontend разработчик",
      "salary": 120000,
      "experience": "3 года",
      "about": "Описание",
      "is_active": 1,
      "Profession": { "id": 1, "name": "Программист" },
      "Applicant": {
        "id": 5,
        "user_id": 1,
        "city": "Москва",
        "created_at": "...",
        "User": { "id": 1, "first_name": "Иван", "last_name": "Иванов", "email": "..." }
      }
    }
  ],
  "total": 50,
  "page": 1,
  "totalPages": 10
}
```

## Вакансии

### `GET /api/vacancies`

Поиск и фильтрация активных вакансий (`is_active = 1`). Пагинация.

**Query параметры:**

| Параметр | Тип | Описание |
| ---------- | ----- | ----------- |
| profession_id | int | ID профессии |
| city | string | Город |
| experience | string | Требуемый опыт (например "1-3 года") |
| salary_from | int | Нижняя граница зарплаты (сравнивается с `salary_to`) |
| salary_to | int | Верхняя граница (сравнивается с `salary_from`) |
| employment_type | string | Тип занятости (на русском: "Полная занятость" и т.д.) |
| search | string | Поиск по заголовку, описанию, опыту, типу, городу, компании, профессии |
| page | int | Номер страницы (default 1) |
| limit | int | Записей на странице (default 5) |

**Успешный ответ (200):**

```json
{
  "success": true,
  "vacancies": [
    {
      "id": 20,
      "title": "React разработчик",
      "description": "...",
      "salary_from": 100000,
      "salary_to": 150000,
      "city": "Москва",
      "employment_type": "full-time",
      "employment_type_display": "Полная занятость",
      "experience_required": "1-3 года",
```

## Примечания

- Все даты возвращаются в формате ISO 8601 (`"2025-01-01T00:00:00.000Z"`), но используются в сокращенном формате `"гггг-мм-дд чч-мм"`.
- Статусы `is_active`:
  - `0` – заблокирован / неактивен
  - `1` – активен
  - `2` – ожидает верификации (для новых вакансий, резюме, пользователей-работодателей)
  - `3` – другой статус (например, отклонён)
- Для пагинации используются параметры `page` (начиная с 1) и `limit`.
- Поиск (`search`) работает как полнотекстовый по нескольким полям (зависит от эндпоинта).
