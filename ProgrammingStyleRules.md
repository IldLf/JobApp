# Правила написания кода
## Python
**Стандарт: PEP 8 — Style Guide for Python Code**

### Оформление кода
- Отступы: Пробелы - самый предпочтительный метод отступов.
- Табуляция должна использоваться только для поддержки кода, написанного с отступами с помощью табуляции.
- Длина строки: Максимальная длина строки — 79 символов.
- Пустые строки: Отделяйте функции и классы двумя пустыми строками. Методы внутри класса отделяйте одной пустой строкой.
- Импорты: Каждый импорт на отдельной строке. Импорты всегда ставятся вверху файла. Группируйте их в порядке:
1. Импорты из стандартной библиотеки.
2. Импорты сторонних библиотек.
3. Импорты внутренних модулей вашего проекта.

### Переменные и функции: 
- snake_case (слова через нижнее подчеркивание). Пример: get_user_name, total_count.
- Классы: CapitalizedWords (или PascalCase). Пример: class UserProfile:.
- Константы: UPPER_SNAKE_CASE. Пример: MAX_RETRIES = 5.
- Приватные переменные/методы: Начинаются с одного нижнего подчеркивания _internal_method.

### Пробелы:
- Ставьте пробелы вокруг операторов присваивания, сравнения и арифметических операций (=, ==, +, -, *, /).
Например: x = 1; y = x + 2
- Не ставьте пробелы перед запятой, двоеточием или точкой с запятой: print(x, y), а не print(x , y).

## HTML
**Стандарт: W3C HTML Living Standard / Google HTML Style Guide**

### Синтаксис и форматирование
- Тип документа: Объявление DOCTYPE должно быть первым элементом: !DOCTYPE html.
- Кодировка: Кодировка UTF-8 должна быть объявлена в первой строке head: meta charset="UTF-8".
- Регистр символов: Все имена тегов и атрибутов должны быть в нижнем регистре.
- Кавычки: Значения атрибутов должны заключаться в двойные кавычки " ". Использование одинарных кавычек не допускается.
- Закрытие тегов: Все теги, кроме самозакрывающихся, должны иметь явный закрывающий тег.
- Отступы: Для вложенных элементов используется отступ в 2 пробела. Это обеспечивает визуальную иерархию документа.

### Семантика 
- Семантические элементы: Должны использоваться элементы HTML5, описывающие структуру и содержание: header, nav, main, article, section, aside, footer.
- Альтернативный текст: Все элементы <img> должны содержать атрибут alt с соответствующим описанием изображения для обеспечения доступности (WCAG).
- Логическая структура: Заголовки должны использоваться последовательно и отражать иерархию содержания.

## JavaScript
**Стандарт: Airbnb JavaScript Style Guide / Google JavaScript Style Guide / ECMAScript Specifications**

### Форматирование и структура
- Точка с запятой: Использование точки с запятой в конце каждой инструкции обязательно.
- Отступы: 2 пробела на один уровень отступа.
- Длина строки: Максимальная длина строки — 80 символов.
- Кавычки: Предпочтительно использование одинарных кавычек ' ' для строк. Двойные кавычки допускаются для JSON и строк, содержащих одинарные кавычки.
- Фигурные скобки: Открывающая фигурная скобка размещается на той же строке, что и оператор (стиль "egyptian brackets").

### Именование
- Переменные и функции: camelCase (первое слово со строчной, последующие с заглавной). Пример: let userName, function getUserData().
- Классы и конструкторы: PascalCase. Пример: class UserProfile.
- Константы: UPPER_SNAKE_CASE для глобальных констант, известных до начала выполнения. Пример: const MAX_RETRY_COUNT = 3.
- Приватные члены: Именование с ведущим подчеркиванием _privateMethod.

### Объявления переменных
- Использование const для переменных, которым не требуется переназначение.
- Использование let для переменных, которым требуется переназначение.
- Использование var не рекомендуется.

## CSS
**Стандарт: Airbnb CSS / Sass Style Guide / Google CSS Style Guide**

### Синтаксис и форматирование
- Отступы: 2 пробела для каждого уровня отступа.
- Правила форматирования: Каждое объявление должно располагаться на новой строке. Открывающая фигурная скобка { должна находиться на той же строке, что и селектор, с пробелом перед ней.
- Пробелы: После двоеточия : обязательно ставится пробел. Перед двоеточием пробел не ставится.
- Точка с запятой: Каждое объявление должно заканчиваться точкой с запятой ;, включая последнее объявление в блоке.

### Единицы измерения
Предпочтительно использование относительных единиц (rem, em, %, vh, vw) вместо абсолютных (px) для обеспечения доступности и адаптивности интерфейсов.
